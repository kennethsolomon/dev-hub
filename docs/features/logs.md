# Logs

> **Status:** Complete
> **Web:** Complete

## Overview

DevHub captures stdout and stderr from running services into per-run log files and streams them in real-time via Server-Sent Events (SSE). Each run gets a unique log file at `data/logs/{runId}.log`.

## Database Schema

Logs are stored as files, not in the database. The `runs` table tracks the log file path:

| Column | Notes |
|--------|-------|
| `runs.log_path` | Absolute path to log file for this run |
| `runs.id` | UUID used as log filename |

## Business Logic

### Log Writing

In `ProcessManager.startService()`:
1. Log directory created at `data/logs/` (recursive mkdir)
2. `WriteStream` opened for the run's log file
3. stdout and stderr piped through `writeLog()` which adds timestamps and stream labels:
   ```
   [2026-01-01T00:00:00.000Z] [stdout] Server started on port 3000
   [2026-01-01T00:00:00.000Z] [stderr] Warning: deprecated API
   ```
4. Each log line also emitted as `'log'` event on the ProcessManager EventEmitter

### SSE Streaming

`GET /api/logs/stream` creates a `ReadableStream` that:
1. Subscribes to ProcessManager `'log'` and `'exit'` events
2. Filters by `serviceId` query param if provided
3. Sends JSON-encoded SSE `data:` frames
4. Sends keepalive comments (`: keepalive`) every 15 seconds
5. Cleans up listeners on request abort

### Log Event Shape

```json
{
  "serviceId": "uuid",
  "timestamp": "2026-01-01T00:00:00.000Z",
  "stream": "stdout",
  "text": "Server started on port 3000\n"
}
```

### Exit Event Shape

```json
{
  "serviceId": "uuid",
  "type": "exit",
  "code": 0,
  "signal": null
}
```

### Historical Log Reading

`GET /api/logs/[runId]` reads the log file from disk for completed runs.

## API Contract

### `GET /api/logs/stream`

SSE endpoint for real-time logs.

**Query params:**
- `serviceId` (optional) — filter to single service

**Response:** `200 OK` with `Content-Type: text/event-stream`

### `GET /api/logs/[runId]`

Read historical log file for a completed run.

**Response:** `200 OK` with log file contents

## Permissions & Access Control

Same as projects — requires auth if enabled.

## Edge Cases

- **Stream closed by client:** Catches enqueue errors silently, cleans up listeners on abort
- **Service exits while streaming:** Exit event sent to all connected clients
- **No running services:** Stream stays open with keepalive, sends data when services start
- **Large log files:** Historical endpoint reads entire file (no pagination in V1)

## Error States

| Scenario | Behavior |
|----------|----------|
| Log file not found | Returns error for historical endpoint |
| Stream controller closed | Silently catches enqueue error |

## Platform Notes

- SSE uses Web Streams API (Next.js App Router compatible)
- Keepalive interval prevents proxy/load balancer timeouts
- Log files persist on disk at `data/logs/` (gitignored)
